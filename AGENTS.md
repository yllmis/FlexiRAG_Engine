# AGENTS 执行约定（FlexiRAG Engine）

本文档用于约束本仓库中 AI 编码助手的默认执行行为。

## 1. 语言与沟通

- 所有回复、注释、说明统一使用中文。
- 每次代码改动后必须明确说明：
  - 修改了哪些文件
  - 为什么修改
  - 验证结果

## 2. 改动触发规则

- 默认只提供方案与建议，不直接改代码。
- 仅当用户明确表达类似“修改代码 / 写代码 / 直接改”时，才执行文件改动。

## 3. 环境与命令

- 按 WSL/Linux 思路优先提供路径与命令。
- 遇到服务启动失败，优先检查容器、端口、环境变量与数据库扩展状态。

## 4. 测试与验收（强约束）

- 不能“强行通过”，必须通过真实可执行检验验证逻辑与架构。
- 每次改动后至少执行：
  - `go test ./...`
- 若当前改动缺少对应测试文件，必须补充单测。
- 测试文件命名规则：按被测文件命名（例如 `handler.go` 对应 `handler_test.go`），不要按单个方法命名测试文件。

## 5. 提交与推送

- 提交信息使用中文规范前缀：`feat:`、`fix:`、`release:`、`chore:` 等。
- 推送前需先说明本次变更范围，再执行推送。

## 6. API/架构改动约定

- 变更遵循分层：Router -> Handler -> Repository/Port。
- 先补 Port 接口，再补基础设施实现，再暴露 Handler 与 Router。
- RESTful 风格优先：
  - 列表查询使用 `GET /resources`
  - 局部更新使用 `PUT /resources/:id`

## 7. 文档维护

- 当用户新增执行约束时，应同步更新本文档。
- 若规则冲突，以用户最新明确要求为准。
